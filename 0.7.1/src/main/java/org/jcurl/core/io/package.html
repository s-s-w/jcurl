Stream related IO / serialisation.

Serialisation

There are several aspects to this topic as there are different things to store and retrieve, namely

  • rock setup (locations and velocities) as e.g. immediately after a player has released a rock.
  • trajectories - what happens after the release. These come (at least) in 2 flavours:
    • Analytic, described by curves
    • Discrete, described by incremental rock positions
  • Event, game, end and release meta data (e.g. where, when, what, who)
  • Ice properties as e.g. friction and curl.
  • Scores
