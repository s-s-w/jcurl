Generic math helpers without dependencies to other jcurl packages. Could even be a separate binary - therefore it's package name is not org.jcurl.core.math

Why isn't math framework X used?

First of all every framework is likely to bring unneeded/unwanted stuff but this core library should remain slim anf fast, concerning both download and runtime. It should be useable within applets or java webstart over slow networks. Therefore it also has to have as few dependencies as possible.

Second the requirements here are rather simple. R^1 -> R^3 monotone, C1 continuous functions. That's all.

Third I use a dirty trick to pack up 2D and angular position/speed into one 3D {@link org.jcurl.core.api.Rock}. This contradicts the JSR-275 measure concept, so jscience will not like this.

But nevertheless the following frameworks were examined:

Jakarta Commons Math

  • brings quite a bit unneeded stuff (jar of 331K),
  • Exceptions on function evaluation,
  • getting derivatives seems a bit a hassle (via the regular Interfaces),
  • No interfaces for multi-dimensional functions

JScience

  • brings a lot of unneeded stuff (jar of 680K),
  • heavy Boxing/Unboxing because of generics (maybe this isn't critical?),
  • evaluating a functions changes inner state and therefore isn't threadsafe.

I admit that jscience is really tempting. I'll keep an eye on it.
